# 6×6 リバーシ

6×6 盤のリバーシに「マーク石とのコンボによるダメージ」「HP 制の勝敗」を載せた攻略・検討用の Web サイトです。

詳細な仕様は **[docs/SPEC.md](docs/SPEC.md)** を参照してください。

## 起動方法

Web Worker を使うため、`file://` ではなく HTTP サーバー経由で開いてください。

```sh
python3 -m http.server 8000
# ブラウザで http://localhost:8000/ を開く
# （コンテナ利用時はホストから `make open`）
```

## ルール

- 6×6 盤の標準リバーシ。黒が先手。
- 初期盤面（中央4石）以降に打たれた石には**マーク**が付く（黒石は 🐱、白石は 🐶）。**反転されるとマークは消える**（復活しない）。
- **打った石と自分のマーク石で相手の石を挟んで返すとコンボ成立**。1 手の獲得ポイントは**ポイント式 f(R, C, D)** で決まる（R = 返した枚数、C = コンボ数 = マーク石で挟んだライン数、D = 確定コンボなら 1）。
  - **確定コンボ（D=1）**: 打った石が相手の次の 1 手では絶対に返されず、かつ相手がどう応じても次の自分の手番でその石を起点にコンボできる場合（= 次のコンボが約束された手）。
  - **D はその手で打った石だけが対象**（以前から置かれている石は対象外）。
  - カスタム式も入力可能。**黒と白で別々の式を設定できる**。式を変更すると過去の手も含めて全ポイントが再計算される。
- 両プレイヤーは**初期 HP**（デフォルト各 30000、黒白別に画面で変更可）を持ち、獲得ポイント = **相手への与ダメージ**。
  **相手の HP を先に 0 にした方の勝ち**。盤面が埋まる／両者連続パスの場合は残り HP が多い方の勝ち。
- 打てる場所がなければ自動パス（与ダメ 0）。

## 画面の見方

- **黄色マス**: 次に打てる場所 / **橙マス**: その手でコンボが成立する場所（右上に与ダメージ、下部に R/C/D を表示）
- **青枠マス**: 解析エンジンの推奨手 / **青枠の石**: 直前の着手
- 「最善プレイ解析」: 双方最善時の最終 HP 差（黒−白）、または KO による勝敗確定を表示

## 実装メモ

| ファイル | 役割 |
|---|---|
| `engine.js` | ゲームロジック + 探索（UI と Worker で共有） |
| `worker.js` | 探索専用 Web Worker |
| `game.js` | UI 制御（描画・待った・自動パス・解析依頼） |

探索は **negamax + αβ枝刈り + 置換表 + 反復深化**。評価値は「手番側から見た双方最善時の
最終 HP 差」で、与ダメが相手の残り HP 以上になる手（KO）は 勝ち確定値 + 最終 HP 差 として
扱う（黒の KO では白の同一ラウンドの最善応手を受けた 2 ターン分で計算）。
状態（盤面 2bit×36 + マーク 1bit×36 + 手番/パス）を 9 文字の文字列キーにパックして
置換表を引く（HP は置換表エントリ側で一致確認）。読みの地平はゲーム開始から
16 手目に固定（深さ = 16 − 経過手数、時間上限なし）。D（確定コンボ）の判定は
「相手の応手で変化し得るのは各レイの停止マスだけ」という性質を使い、相手の
全手生成なしで候補手だけを検証する。探索は Web Worker 上で走り、反復深化の
途中経過を随時表示する。詳細は [docs/SPEC.md](docs/SPEC.md)。
